## 用户需求

基于现有 VB.NET CFD 引擎，新增"风洞外流动"模拟功能模块：解析三维体素模型 JSON 文件（airplane1_28x9x32_voxels.json），将其加载为引擎计算空间（模型本体作为固体障碍、外部作为流体），定义水平向右（+X 方向）的均匀来流场，模拟气流掠过模型表面的流动，并量化尾流、涡旋等湍流特征物理结果；最后在 test/ 项目中新增测试代码文件验证该模块。

## 产品概述

在发酵罐搅拌模拟之外，新增一条"外部绕流"模拟链路：体素 JSON → VoxelShape（计算空间真相源）→ 风洞容器（复用现有求解器）→ 湍流指标输出。气流从左向右吹过模型表面，在模型后方形成尾流与涡旋，通过涡量、总涡量（enstrophy）、速度与尾流亏损等指标观测 turbulent-like 物理结果。

## 核心功能

- 解析体素 JSON：读取 schema/grid/data，按 index=(x*height+y)*depth+z 重建三维体素空间，依据 valueSemantics 将值 1=固体、值 0=流体 反转映射为 VoxelShape（True=活动流体、False=固体障碍）。
- 加载进引擎：以 VoxelShape 构建 FluidField，并通过 ToSolidMask 设置 StableFluidsSolver 的固体掩膜。
- 水平向右来流场：左边界定常入流（U=U∞，V=W=0）、右边界零梯度出流、其余四面自由滑移壁面；初始全场置为来流速度。
- 湍流/物理结果观测：计算最大速度与平均速度、涡量 ω=∇×u 及总涡量 enstrophy=Σ|ω|²（涡旋/湍流强度代理指标）、下游尾流速度亏损。
- 测试验证：在 test/ 新增独立代码文件，加载模型、运行模拟、断言/打印上述指标（无 NaN、enstrophy>0、尾流形成等）。

## 技术栈

- 语言/框架：VB.NET（.NET 10，LangVersion 16.9），与现有工程一致
- 数据结构：复用现有 Tensor、FluidField、VoxelShape、StableFluidsSolver
- JSON 解析：复用项目已采用的 System.Text.Json（System.Text.Json.JsonDocument），不引入新依赖
- 测试：复用现有 test/ 控制台工程（无单元测试框架，采用控制台 PASS/FAIL 风格）

## 实现方案

整体策略：在不修改核心求解器与发酵罐逻辑的前提下，新增两个类与一个解析模块——

1. `VoxelModelLoader`：解析体素 JSON 为 `VoxelShape`（含反转映射与元数据）。
2. `WindTunnel`：与 `FermentationTank` 平行的"模拟容器"，复用 `StableFluidsSolver.Advect/Diffuse/Project`，每步投影后覆盖风洞入/出流边界，并 `EnforceSolidMask` 维持模型表面壁面。
3. 通过 `FluidSim.CreateWindTunnel` 工厂暴露，与 `CreateDefault/CreateCapsule` 风格一致。

关键决策与取舍：

- 反转映射：JSON 的 valueSemantics 明确 1=solid 障碍、0=fluid，而 VoxelShape 约定 True=活动流体、False=固体障碍，必须反转（active = data[idx] <> 1），否则模型会被当成流体、外部成障碍。
- 风洞边界而非封闭箱体：每次 `Project` 末端求解器会把 6 面统一设为壁面，故在投影之后强制左面 i=0 入流、右面 i=nx-1 零梯度出流；其余四面沿用求解器自由滑移壁面（NoSlipWalls=False），形成开放来流。
- 初始全场 U=U∞：使来流即时存在，模型扰动后在尾流区发展出涡旋，便于观测湍流特征。
- 湍流指标选用 enstrophy：Stable Fluids 为层流求解器无脉动场，无法直接算 RANS 湍流；以解析速度场的涡量平方积分（enstrophy）作为涡旋/湍流结构强度代理，配合尾流亏损，直观且可计算。
- 性能：网格仅 28×9×32=8064 体素，单步 O(N·JacobiIter) 极快，运行 120 步级在毫秒级，无需并行；内存开销可忽略。

## 实现注意事项

- 复用：VoxelShape.Index 约定 `index=(x*Height+y)*Depth+z` 与 JSON 的 indexFormula 完全一致，无需额外坐标变换；直接复用 `ToSolidMask/EnforceSolidMask`。
- 边界安全：入流仅在流体体素（IsActive）处施加，避免覆盖恰好位于 i=0 的固体体素；出流用零梯度复制内部一层。
- 解析健壮性：用 JsonDocument 读取，校验 `data.Length = width*height*depth`，文件缺失/格式错误抛出含路径的清晰异常。
- 向后兼容：仅新增文件与少量工厂方法，不改动 FermentationTank/Stirrer/StableFluidsSolver 既有逻辑。
- 测试路径：test.vbproj 将 JSON 拷贝到输出目录；测试按文件名在 AppContext.BaseDirectory 解析，支持命令行路径覆盖。

## 架构设计

风洞模块与现有发酵罐模块共享底层求解器与流体场数据结构，二者为并列的"模拟容器"：
`VoxelModelLoader（解析）→ VoxelShape（计算空间真相源）→ WindTunnel（FluidField + StableFluidsSolver，风洞边界 + 湍流指标）`
